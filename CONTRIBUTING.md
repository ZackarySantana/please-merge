# Contributing

Thanks for your interest in contributing.

## Getting started

1. Fork the repository and clone it locally.
2. Open `index.html` in a browser. There is no build step.
3. Make your changes and test them in the browser.

## Submitting changes

1. Create a branch for your change.
2. Keep commits focused and descriptive.
3. Open a pull request against `main` with a brief summary of what changed and why.

## Reporting issues

Open an issue on GitHub. Include what you expected to happen, what actually happened, and steps to reproduce if applicable.

## Code style

- No build tools or frameworks. The project is vanilla HTML, CSS, and JavaScript.
- Keep functions small and state management straightforward.
- Test your changes across at least one modern browser before submitting.
